# 非機能テスト実装チェックリスト

## 目的
このドキュメントは、現時点で未実装の非機能テスト項目を整理し、実装順序を管理するためのチェックリストです。

## スコープ
- 対象: backend API、frontend UI、CI実行テスト
- 観点: 性能、負荷/耐久、可用性、互換性
- 非対象: 機能仕様の正当性のみを検証する単体/結合/E2Eテスト

## 実装チェックリスト

## 1. 性能（レスポンス時間、スループット）
- [x] APIレスポンス時間のSLO検証（p50/p95/p99、上限ms）
- [x] APIスループット検証（RPS、処理件数/秒）
- [x] エンドポイント別の性能回帰テスト（基準値比較）
- [x] 大量データ条件でのDBクエリ性能検証（閾値つき）
- [x] Go Benchmarkテスト（`func Benchmark...`）の追加
- [x] フロントエンドのWeb Vitals/Lighthouse回帰測定

## 2. 負荷/耐久（同時アクセス、長時間）
- [x] 負荷プロファイル試験（通常負荷、ピーク負荷、スパイク）
- [x] 高並列アクセス試験（多数同時ユーザー/セッション）
- [x] 長時間耐久試験（soak/endurance）
- [x] リソース飽和試験（CPU/メモリ/DB接続）
- [x] 負荷ツールシナリオ管理（k6/Locust/JMeter等）
- [x] CIまたは定期ジョブでの負荷試験実行

## 3. 可用性（障害時リカバリ）
- [ ] DB障害時の復旧試験（停止→復帰、再接続）
- [ ] Redis障害時の復旧試験（停止→復帰、再接続）
- [ ] アプリ再起動時の継続性試験（in-flight request含む）
- [ ] フェイルオーバー試験（複数インスタンス想定）
- [ ] readiness/liveness劣化時の挙動試験（依存先断）
- [ ] バックアップ/リストア訓練（RPO/RTO検証）
- [ ] ネットワーク分断/タイムアウト時の回復試験

## 4. 互換性（ブラウザ/端末差）
- [ ] FirefoxでのE2E実行
- [ ] WebKit/SafariでのE2E実行
- [ ] Edge互換実行
- [ ] モバイル端末エミュレーションE2E（iOS/Android）
- [ ] タブレット/画面幅別のE2E検証
- [ ] CIでのブラウザ×端末マトリクス実行

## 推奨実装順序（P0 -> P3）
- [ ] P0: 互換性基盤（Playwright複数ブラウザ化）+ 性能SLO測定基盤
- [ ] P1: APIレスポンス時間/スループット + 高並列負荷
- [ ] P2: 長時間耐久 + 可用性（DB/Redis障害復旧）
- [ ] P3: フェイルオーバー + バックアップ/リストア演習

## 完了条件
- [ ] 4観点の全チェック項目が実行可能なテストとして実装済み
- [ ] 主要項目がCIまたは定期実行で自動検証される
- [ ] 性能/可用性の判定基準（SLO/SLA）がドキュメント化されている
